## 1. 설치 및 실행 가이드

대시보드는 **백엔드 분석 파이프라인과 독립적으로** 실행되는 웹 애플리케이션입니다.  
백엔드(`python -m src.cli run-all`)를 먼저 실행해 분석 결과 JSON 파일을 생성한 후 대시보드를 띄워야 합니다.

---

### macOS

#### 1단계 — 백엔드 환경 구성 (최초 1회)

터미널을 열고 프로젝트 루트 디렉토리로 이동합니다.

```bash
# 프로젝트 루트 이동 (경로는 실제 위치에 맞게 수정)
cd ~/projects/kisdi-platform/news-platform-monitor

# Python 가상환경 생성
python3 -m venv venv

# 가상환경 활성화
source venv/bin/activate

# 의존 패키지 설치
pip install -r requirements.txt

# 설정 파일 생성
cp config.yaml.example config.yaml
```

`config.yaml`을 텍스트 편집기로 열어 API 키를 입력합니다.  
API 키 발급 방법은 [gemini-api-setup.md](gemini-api-setup.md)를 참고하세요.

#### 2단계 — 백엔드 실행 (데이터 수집 & 분석)

```bash
# 가상환경이 활성화된 상태에서 실행
source venv/bin/activate
```

**[방법 A] 전체 프로세스 한 번에 실행**

```bash
python -m src.cli run-all
```

**[방법 B] 단계별 개별 실행**

```bash
python -m src collect --news      # 1. 뉴스 수집
python -m src preprocess          # 2. 뉴스 전처리
python -m src analyze-news        # 3. 뉴스 LLM 분석

python -m src collect --rss       # 4. 보도자료 수집
python -m src analyze-rss         # 5. 보도자료 LLM 분석

python -m src generate-recommendations  # 6. 통합 제언 생성

python -m src.cli status
```

완료되면 `dashboard/public/data/` 폴더에 분석 결과 JSON 파일이 생성됩니다.

#### 3단계 — 대시보드 환경 구성 (최초 1회)

새 터미널 탭을 열거나 같은 창에서 계속 진행합니다.

**Node.js 설치 확인**

```bash
node -v   # 18 이상 권장
npm -v
```

Node.js가 없으면 [https://nodejs.org](https://nodejs.org) 에서 LTS 버전을 설치합니다.  
또는 Homebrew를 사용합니다.

```bash
brew install node
```

**대시보드 의존 패키지 설치**

```bash
cd dashboard
npm install
```

#### 4단계 — 대시보드 실행

```bash
# dashboard 폴더 안에서 실행
npm run dev
```

터미널에 아래와 같은 메시지가 표시되면 성공입니다.

```
  VITE v5.x.x  ready in xxx ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
```

브라우저에서 **http://localhost:5173** 을 열면 대시보드가 표시됩니다.

> 종료하려면 터미널에서 `Ctrl + C`를 누릅니다.

**명령 프롬프트(cmd)** 또는 **PowerShell**을 실행합니다.

```bat
:: 프로젝트 루트 이동 (경로는 실제 위치에 맞게 수정)

python -m venv venv

:: 가상환경 활성화 (cmd)
venv\Scripts\activate.bat

:: 가상환경 활성화 (PowerShell)
:: venv\Scripts\Activate.ps1

:: 의존 패키지 설치
pip install -r requirements.txt

:: 설정 파일 생성
copy config.yaml.example config.yaml
```

> ```
> Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

API 키 발급 방법은 [gemini-api-setup.md](gemini-api-setup.md)를 참고하세요.

### 자주 발생하는 오류

| 증상 | 원인 | 해결 방법 |
|------|------|-----------|
| `npm: command not found` | Node.js 미설치 | nodejs.org에서 LTS 설치 |
| 대시보드가 빈 화면 | 분석 JSON 파일 없음 | 백엔드 `run-all` 먼저 실행 |
| `port 5173 already in use` | 포트 충돌 | `npm run dev -- --port 5174` |
| `pip install` 실패 | Python 버전 낮음 | Python 3.10 이상 사용 |
| PowerShell 스크립트 차단 | 실행 정책 제한 | 위 `Set-ExecutionPolicy` 명령 실행 |

## 2. 리스크 스코어란?

리스크 스코어는 **각 기사가 플랫폼 규제·정책 환경에 미치는 위험 수준**을 0–100점으로 수치화한 값입니다.  
백엔드가 기사 원문을 Google Gemini AI에 전달하면, AI가 내용을 분석하여 아래 기준에 따라 점수를 산출합니다.

### 산출 기준

| 점수 범위 | 수준 | 의미 | 예시 상황 |
|-----------|------|------|-----------|
| 0 – 30 | 관심 | 중립적 정보 제공, 정책 발표 | 새 서비스 출시 보도, 제도 안내 공문, 정책 설명 자료 |
| 31 – 60 | 주의 | 규제 논의, 제도 개선, 조사 착수 | 국회 입법 논의, 공정위 실태조사 착수, 소비자단체 민원 제기 |
| 61 – 80 | 고위험 | 과징금·처분·갈등 본격화 | 공정위 과징금 부과, 방통위 시정명령, 노동청 특별 점검 |
| 81 – 100 | 고위험 | 심각한 법적 제재, 긴급 규제, 소비자 피해 | 검찰 수사 개시, 긴급 영업정지, 대규모 집단 피해 사건 |

> **고위험 기준선은 70점**입니다. 대시보드의 빨간 배지, 히트맵 진한 색상, 클러스터 랭킹 기준선 모두 70점을 임계값으로 사용합니다.

### 공통 UI 요소

#### 날짜 범위 필터 (화면 상단 우측)

| 버튼 | 의미 |
|------|------|
| 전체 | 수집된 모든 기간의 데이터 표시 |
| 최근 7일 | 오늘 기준 7일 이내 데이터만 표시 |
| 최근 30일 | 오늘 기준 30일 이내 데이터만 표시 |
| 최근 90일 | 오늘 기준 90일 이내 데이터만 표시 |

> 필터를 변경하면 모든 섹션의 수치와 차트가 동시에 갱신됩니다.

#### 데이터 갱신 방법

대시보드는 실행 시점의 JSON 파일을 읽어 표시합니다. 백엔드 파이프라인을 다시 실행한 뒤에는 **브라우저를 새로고침(F5 또는 Cmd+R)** 하면 최신 데이터가 반영됩니다.

#### 분석 진행률 표시줄

헤더 하단에 표시되는 `분석 완료 N건 / 전체 N건` 바는 백엔드가 처리한 분석 진행률을 나타냅니다.  
진행률이 100%에 미치지 못하는 경우, 미분석 항목이 남아 있으므로 `python -m src.cli analyze`를 다시 실행하면 됩니다.

### 섹션 1: 개요 (Overview)

정책 이슈 전반의 현황을 한눈에 파악하는 요약 섹션입니다.

#### 요약 카드 (4개)

| 카드 | 의미 |
|------|------|
| **전체 보도자료** | 선택한 기간 내 수집된 보도자료 및 뉴스 기사의 총 건수 |
| **분석 완료** | LLM(Gemini) 분석이 성공적으로 완료된 기사 건수 |
| **평균 리스크** | 분석된 기사들의 리스크 스코어 평균값 (0–100점). 70점 이상이면 빨간색으로 강조 표시 |
| **고위험 이슈** | 리스크 스코어 70점 이상인 기사의 건수 |

#### 정책영역별 기사 수 (막대 차트)

수집된 기사가 7개 정책 영역 중 어디에 얼마나 분포하는지 보여줍니다.

| 정책 영역 | 의미 |
|-----------|------|
| 공정거래 | 독점·시장 지배력 남용·불공정 거래 관련 이슈 |
| 소비자보호 | 환불·피해 구제·소비자 권리 관련 이슈 |
| 개인정보 | 개인정보 수집·처리·유출·보호 관련 이슈 |
| 노동 | 플랫폼 종사자 처우·노동권 관련 이슈 |
| 콘텐츠/저작권 | 콘텐츠 유통·저작권 침해 관련 이슈 |
| 안전 | 제품·서비스 안전사고 관련 이슈 |
| AI/자동화 | 인공지능·자동화·알고리즘 관련 이슈 |

> 하나의 기사가 여러 정책 영역에 동시에 분류될 수 있으므로, 합계가 전체 기사 수를 초과할 수 있습니다.

#### 감성 분포 (도넛 차트)

기사의 전반적인 논조(감성)를 세 가지로 구분해 비율로 보여줍니다.

| 색상 | 감성 | 의미 |
|------|------|------|
| 초록 | 긍정 | 플랫폼·정책에 우호적이거나 문제 해소를 다룬 기사 |
| 빨강 | 부정 | 비판·문제 제기·피해 사례 등 부정적 논조의 기사 |
| 회색 | 중립 | 사실 전달 위주로 논조가 명확하지 않은 기사 |

#### 리스크 히트맵 (플랫폼 × 정책영역)

행(플랫폼)과 열(정책영역)이 교차하는 셀의 색상 농도로 리스크 수준을 나타냅니다.

- **색상이 진할수록** 해당 플랫폼 + 정책영역 조합의 평균 리스크가 높습니다.
- 셀 안의 숫자는 해당 조합의 평균 리스크 스코어(0–100)입니다.
- **행(플랫폼)을 클릭**하면 섹션 4의 플랫폼 분석이 해당 플랫폼으로 필터링됩니다.
- 플랫폼은 국내(네이버·카카오·쿠팡 등)와 해외(구글·메타·틱톡 등)로 구분됩니다.

### 섹션 2: 이슈 트렌드

시간 흐름에 따른 이슈 변화를 추적하는 섹션입니다.

#### 일별 보도자료 수 (선 그래프)

X축은 날짜(MM-DD), Y축은 기사 건수입니다.  
특정 날짜에 건수가 급증한 경우, 해당 일에 플랫폼 관련 주요 사건이나 정책 발표가 있었을 가능성이 높습니다.

#### 이슈 클러스터 리스크 랭킹

자주 등장한 키워드를 기준으로 묶인 이슈 클러스터의 리스크를 순위별로 보여줍니다.

| 요소 | 의미 |
|------|------|
| 키워드명 (좌측) | 해당 이슈 클러스터를 대표하는 키워드 |
| 막대 길이 | 평균 리스크 스코어 (길수록 위험도 높음) |
| 막대 색상 | 리스크 수준 (빨강 ≥70, 주황 50–69, 파랑 <50) |
| 숫자 배지 (우측) | 해당 키워드가 등장한 기사 건수 |
| 빨간 점선 | 고위험 기준선 (70점) |

### 섹션 3: 키워드 & 이슈 목록

주요 키워드와 개별 이슈를 탐색하는 섹션입니다.

#### 키워드 태그 클라우드

분석된 기사에서 추출된 상위 30개 키워드를 시각화합니다.

| 시각적 요소 | 의미 |
|-------------|------|
| 글자 크기 | 키워드 등장 빈도 (클수록 더 자주 언급됨) |
| 글자 색상 | 해당 키워드가 주로 속한 정책 영역 색상 |
| 클릭 | 해당 키워드가 포함된 기사만 하단 이슈 목록에 필터링 표시 |

#### 이슈 목록 (테이블)

각 기사의 상세 정보를 표 형태로 보여줍니다.

| 열 | 의미 |
|----|------|
| 날짜 | 기사 발행 또는 보도자료 등록일 |
| 출처 | **보도자료** (정부 기관 RSS) 또는 **뉴스** (네이버 뉴스 API) |
| 제목 | 기사 제목 (클릭 시 원문 링크로 이동) |
| 리스크 | 0–100점 리스크 스코어. 색상 배지로 수준 구분 |
| 감성 | LLM이 판단한 기사 논조 (긍정/부정/중립) |
| 정책영역 | 해당 기사가 분류된 정책 영역 태그 |

**정렬 기준**
- 기본값은 **리스크 높은 순**으로 정렬됩니다.
- 날짜순 정렬로 전환할 수 있습니다.

**표시 조건**
- 키워드 클라우드에서 특정 키워드를 클릭하면 해당 키워드 관련 기사만 표시됩니다.
- 필터가 없을 경우 리스크 70점 이상 고위험 기사가 우선 표시됩니다.

### 섹션 4: 플랫폼 분석 & AI 정책 제언

개별 플랫폼의 이슈 현황과 AI가 생성한 정책 대응 방향을 제공하는 섹션입니다.

#### 플랫폼별 감성 분포 (수평 누적 막대 차트)

기사 건수 상위 7개 플랫폼의 감성 분포를 누적 막대로 비교합니다.

| 요소 | 의미 |
|------|------|
| 빨간 영역 | 부정 기사 비율 (%) |
| 회색 영역 | 중립 기사 비율 (%) |
| 초록 영역 | 긍정 기사 비율 (%) |
| 우측 숫자 | 해당 플랫폼의 전체 기사 건수 |

부정 비율이 높은 플랫폼일수록 현재 규제·여론 리스크가 높습니다.

#### 플랫폼별 이슈 카드

국내 플랫폼(13개)·해외 플랫폼(6개)·기타로 분류된 각 플랫폼의 이슈 요약 카드입니다.

| 카드 요소 | 의미 |
|-----------|------|
| 상단 색상 바 | 리스크 수준 (빨강: 고위험, 주황: 주의, 파랑: 관심) |
| 리스크 배지 | **고위험** (≥70) / **주의** (50–69) / **관심** (<50) |
| 기사 수 | 해당 플랫폼이 언급된 기사 건수 |
| 최고 리스크 | 해당 플랫폼 관련 기사 중 가장 높은 리스크 스코어 |
| 키워드 태그 | 해당 플랫폼 관련 기사에서 자주 등장한 상위 5개 키워드 |
| 요약 텍스트 | LLM이 생성한 해당 플랫폼 이슈 요약 |

#### AI 정책 제언 카드

Gemini API가 분석 결과를 바탕으로 생성한 정책 대응 방향입니다.

- 각 카드는 하나의 정책 제언 항목을 나타냅니다.
- **제목**: 제언의 핵심 방향 (예: "플랫폼 수수료 투명성 강화 방안")
- **본문**: 제언의 배경·근거·구체적 조치 내용

> AI 정책 제언은 분석된 기사 내용을 기반으로 자동 생성되며, 참고 자료로 활용하되 반드시 전문가 검토를 거쳐야 합니다.

## 4. 색상 코드 체계

### 리스크 수준

| 색상 | 범위 | 의미 |
|------|------|------|
| 빨강 | 70 – 100 | **고위험**: 즉각적인 정책 대응 또는 모니터링 강화 필요 |

| 파랑/회색 | 0 – 49 | **관심**: 현황 파악 수준의 모니터링 유지 |
